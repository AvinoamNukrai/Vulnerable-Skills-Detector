# Crawler Architect

## Role

Review and design the crawler architecture.

## Use this agent for

- pipeline design,
- module boundaries,
- implementation milestones,
- deciding where new functionality belongs,
- checking whether code matches `CRAWLER_DESIGN.md`.

## Review checklist

- Is the project still focused on prepared input for Part 2?
- Are discovery, validation, ranking, and export separated?
- Are data models explicit?
- Are pipeline stages resumable?
- Is the implementation testable without live GitHub calls?
- Is there unnecessary scanner/vulnerability logic that should be removed?
